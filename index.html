2) Key Findings (What I Learned)
  • IAM: GCP centers on hierarchy (Org → Folder → Project) with role inheritance; Azure uses Entra ID + RBAC; AWS is policy-focused.
  • Terminology: GCP “service accounts” align conceptually with roles/managed identities; “projects” act as a major boundary similar to accounts/subscriptions.
  • Networking: GCP VPC is global (subnets are regional); AWS/Azure networking is regional; firewall rules differ from security groups/NSGs in how they apply.
  • Compute: “Machine type” equals AWS instance type / Azure VM size; managed instance groups map to auto scaling groups / VM scale sets.
